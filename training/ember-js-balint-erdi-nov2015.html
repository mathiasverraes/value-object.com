Course Brief

If you have been looking to level up your Ember knowledge (or jumpstart it), this is the perfect time to do it.

We'll build an application from the ground up, gently introducing the main building blocks of the most extensive client-side framework. This gradual approach will help you grok new concepts more easily, building on top of the knowledge you've just acquired.

You might have heard that Ember has a steep learning curve. Let me assure you that understanding a few key concepts crushes the learning curve, and the time of your journey to Ember mastery is largely diminished. This training largely facilitates this.

We'll build a music catalog app, where one can create bands and songs, edit the band's description, rate the songs, and so on. We'll build the app from a very modest start (only showing a list of bands) and then go adding features and using a particular Ember building block or concept to make it ever richer (and learn what they are good for in the process). Here are the rough steps we'll go through*:

  • List the bands like we were in 1999, a barebones bullet list (routing & templates)
  • List the bands and songs on two different URLs, completely independent of each other
  • List the songs related to the selected band (nested routes and UI)
  • Create new bands (actions)
  • Rate the songs with a stellar star rating component (components)
  • Show band info if it exists, songs otherwise, redirect from one route to another, set document title (advanced routing)
  • Create a new song in two steps, implementing a simplistic on-boarding process (controllers)
  • Persist our data and use a real backend (models & Ember Data, JSON API)
  • Sort and search existing songs (query parameters)
  • Safeguard the main user scenarios by writing acceptance tests
  • Make sure the building blocks of our app work as they should (integration and unit tests)
  • Deploy our application to Divshot
  • Build it like it's 2015! (convert our app to ES2015)
  • Less code means less errors (computed macros)
  • Add a couple of animations for better user ergonomics, or just for fluff
  • "I want my own bands!" (authentication & authorization)
  • Another way to structure your code: pods

The app will communicate with a backend using the JSON API format. As this is an Ember.js training, we'll use the API as a black box, with a specification of what should go in and what we expect to come out.

Is this training for you?

The training is for Ember beginners, so if you feel comfortable writing Ember apps and experience the joy of working with the framework, and not against it, this training is probably not for you. On the other hand, if you

  • are just starting out with Ember, or
  • hacked on something and acquired some knowledge but feel like your understanding is spotty, or
  • feel like you could benefit from someone showing you the ropes, and clarifying the concepts,

then this training will suit you well.
